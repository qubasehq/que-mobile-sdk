# Dynamic Tools

Dynamic tools allow the agent to generate and execute custom functionality at runtime, extending its capabilities beyond the built-in action types.

## Overview

The QUE Mobile SDK includes a dynamic tool system that enables:
- **Runtime tool generation**: Create new tools during task execution
- **Custom functionality**: Extend agent capabilities without code changes
- **Tool registry**: Manage and reuse custom tools
- **MCP integration**: Connect to Model Context Protocol servers

## Basic Concepts

### What are Dynamic Tools?

Dynamic tools are custom functions that:
1. Are defined at runtime (not compile-time)
2. Can be generated by the LLM
3. Have typed parameters and return values
4. Can be reused across tasks

### When to Use Dynamic Tools

Use dynamic tools when you need to:
- Perform calculations or data transformations
- Integrate with external APIs
- Add custom business logic
- Extend agent capabilities without rebuilding

## Generating Tools

### Using generate_tool Action

The agent can generate tools using the `generate_tool` action:

```typescript
import { useAgent } from 'que-mobile-sdk';

function DynamicToolExample() {
  const { execute } = useAgent({
    apiKey: process.env.GEMINI_API_KEY!,
  });

  const handleGenerateTool = async () => {
    const task = `
      1. Generate a tool called "calculate_tip" that:
         - Takes parameters: amount (number), percentage (number)
         - Calculates tip amount
         - Returns the tip value
      2. Execute the tool with amount=50, percentage=20
      3. Report the result
    `;
    
    await execute(task);
  };

  return (
    <Button title="Generate & Use Tool" onPress={handleGenerateTool} />
  );
}
```

### Tool Definition Format

Tools are defined with:

```typescript
interface ToolDefinition {
  name: string;                    // Unique tool name
  description: string;             // What the tool does
  parameters: Record<string, {     // Parameter definitions
    type: string;                  // Parameter type
    description?: string;          // Parameter description
    required?: boolean;            // Whether required
  }>;
  execute: (params: any) => Promise<any>; // Execution function
}
```


## Tool Registry

### Registering Custom Tools

Register tools that the agent can discover and use:

```typescript
import { ToolRegistry } from 'que-mobile-sdk';

// Register a custom tool
ToolRegistry.register({
  name: 'convert_currency',
  description: 'Convert amount from one currency to another',
  parameters: {
    amount: {
      type: 'number',
      description: 'Amount to convert',
      required: true,
    },
    from: {
      type: 'string',
      description: 'Source currency code (e.g., USD)',
      required: true,
    },
    to: {
      type: 'string',
      description: 'Target currency code (e.g., EUR)',
      required: true,
    },
  },
  execute: async (params) => {
    // Your conversion logic here
    const rate = await fetchExchangeRate(params.from, params.to);
    const converted = params.amount * rate;
    return {
      success: true,
      result: converted,
      message: `${params.amount} ${params.from} = ${converted} ${params.to}`,
    };
  },
});
```

### Using Registered Tools

Once registered, the agent can use tools in tasks:

```typescript
const { execute } = useAgent({ apiKey: process.env.GEMINI_API_KEY! });

await execute("Convert 100 USD to EUR using the convert_currency tool");
```

### Listing Available Tools

```typescript
import { ToolRegistry } from 'que-mobile-sdk';

// Get all registered tools
const tools = ToolRegistry.list();
console.log('Available tools:', tools.map(t => t.name));

// Get specific tool
const tool = ToolRegistry.get('convert_currency');
console.log('Tool:', tool);
```

### Unregistering Tools

```typescript
import { ToolRegistry } from 'que-mobile-sdk';

// Unregister a tool
ToolRegistry.unregister('convert_currency');

// Clear all tools
ToolRegistry.clear();
```

## Dynamic Tool Generator

### Automatic Tool Generation

The `DynamicToolGenerator` creates tools from natural language descriptions:

```typescript
import { DynamicToolGenerator } from 'que-mobile-sdk';

const generator = new DynamicToolGenerator({
  apiKey: process.env.GEMINI_API_KEY!,
});

// Generate tool from description
const tool = await generator.generate({
  name: 'calculate_discount',
  description: 'Calculate discounted price given original price and discount percentage',
  parameters: {
    price: 'number',
    discount: 'number',
  },
});

// Use the generated tool
const result = await tool.execute({ price: 100, discount: 20 });
console.log('Discounted price:', result); // 80
```

### Tool Generation Process

1. **Description Analysis**: LLM analyzes the tool description
2. **Code Generation**: LLM generates implementation code
3. **Validation**: Generated code is validated for safety
4. **Registration**: Tool is registered in the registry
5. **Execution**: Tool can be executed with parameters

## Executing Dynamic Tools

### Using execute_dynamic_tool Action

Execute a previously generated or registered tool:

```typescript
const { execute } = useAgent({ apiKey: process.env.GEMINI_API_KEY! });

const task = `
  1. Execute the calculate_tip tool with amount=75, percentage=18
  2. Report the tip amount
`;

await execute(task);
```

### Direct Execution

Execute tools directly without the agent:

```typescript
import { ToolRegistry } from 'que-mobile-sdk';

const tool = ToolRegistry.get('calculate_tip');
if (tool) {
  const result = await tool.execute({ amount: 75, percentage: 18 });
  console.log('Tip:', result);
}
```

## MCP Tool Adapter

### Connecting to MCP Servers

The `MCPToolAdapter` allows integration with Model Context Protocol servers:

```typescript
import { MCPToolAdapter } from 'que-mobile-sdk';

// Connect to MCP server
const adapter = new MCPToolAdapter({
  serverUrl: 'http://localhost:3000/mcp',
  apiKey: 'your-mcp-api-key',
});

// Discover available tools
const tools = await adapter.discoverTools();
console.log('MCP Tools:', tools);

// Register MCP tools
for (const tool of tools) {
  ToolRegistry.register(tool);
}
```

### Using MCP Tools

Once registered, MCP tools work like any other tool:

```typescript
const { execute } = useAgent({ apiKey: process.env.GEMINI_API_KEY! });

await execute("Use the weather_forecast tool to get weather for New York");
```

## Advanced Usage

### Tool Composition

Combine multiple tools:

```typescript
ToolRegistry.register({
  name: 'calculate_total_with_tip',
  description: 'Calculate total bill including tip',
  parameters: {
    amount: { type: 'number', required: true },
    tipPercentage: { type: 'number', required: true },
    taxPercentage: { type: 'number', required: true },
  },
  execute: async (params) => {
    // Use other tools
    const tipTool = ToolRegistry.get('calculate_tip');
    const taxTool = ToolRegistry.get('calculate_tax');
    
    const tip = await tipTool.execute({
      amount: params.amount,
      percentage: params.tipPercentage,
    });
    
    const tax = await taxTool.execute({
      amount: params.amount,
      percentage: params.taxPercentage,
    });
    
    return {
      subtotal: params.amount,
      tip: tip.result,
      tax: tax.result,
      total: params.amount + tip.result + tax.result,
    };
  },
});
```

### Conditional Tool Execution

```typescript
ToolRegistry.register({
  name: 'smart_calculator',
  description: 'Performs calculation based on operation type',
  parameters: {
    operation: { type: 'string', required: true },
    a: { type: 'number', required: true },
    b: { type: 'number', required: true },
  },
  execute: async (params) => {
    switch (params.operation) {
      case 'add':
        return params.a + params.b;
      case 'subtract':
        return params.a - params.b;
      case 'multiply':
        return params.a * params.b;
      case 'divide':
        return params.a / params.b;
      default:
        throw new Error(`Unknown operation: ${params.operation}`);
    }
  },
});
```

### Async Tool Execution

Tools can perform async operations:

```typescript
ToolRegistry.register({
  name: 'fetch_user_data',
  description: 'Fetch user data from API',

```


## Error Handling

### Tool Execution Errors

Handle errors in tool execution:

```typescript
ToolRegistry.register({
  name: 'safe_divide',
  description: 'Safely divide two numbers',
  parameters: {
    a: { type: 'number', required: true },
    b: { type: 'number', required: true },
  },
  execute: async (params) => {
    try {
      if (params.b === 0) {
        return {
          success: false,
          error: 'Division by zero',
        };
      }
      return {
        success: true,
        result: params.a / params.b,
      };
    } catch (error) {
      return {
        success: false,
        error: error.message,
      };
    }
  },
});
```

### Validation

Validate parameters before execution:

```typescript
ToolRegistry.register({
  name: 'send_email',
  description: 'Send an email',
  parameters: {
    to: { type: 'string', required: true },
    subject: { type: 'string', required: true },
    body: { type: 'string', required: true },
  },
  execute: async (params) => {
    // Validate email format
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!emailRegex.test(params.to)) {
      return {
        success: false,
        error: 'Invalid email address',
      };
    }
    
    // Send email
    await sendEmail(params);
    return {
      success: true,
      message: 'Email sent successfully',
    };
  },
});
```

## Security Considerations

### Sandboxing

Tools should be sandboxed to prevent malicious code:

```typescript
import { DynamicToolGenerator } from 'que-mobile-sdk';

const generator = new DynamicToolGenerator({
  apiKey: process.env.GEMINI_API_KEY!,
  sandbox: true, // Enable sandboxing
  allowedModules: ['math', 'string'], // Whitelist modules
});
```

### Input Validation

Always validate tool inputs:

```typescript
ToolRegistry.register({
  name: 'process_data',
  description: 'Process user data',
  parameters: {
    data: { type: 'object', required: true },
  },
  execute: async (params) => {
    // Validate input
    if (!params.data || typeof params.data !== 'object') {
      throw new Error('Invalid data format');
    }
    
    // Sanitize input
    const sanitized = sanitizeInput(params.data);
    
    // Process
    return processData(sanitized);
  },
});
```

### Rate Limiting

Implement rate limiting for tools:

```typescript
const rateLimiter = new Map();

ToolRegistry.register({
  name: 'api_call',
  description: 'Call external API',
  parameters: {
    endpoint: { type: 'string', required: true },
  },
  execute: async (params) => {
    const key = `api_call_${params.endpoint}`;
    const lastCall = rateLimiter.get(key) || 0;
    const now = Date.now();
    
    // Rate limit: 1 call per second
    if (now - lastCall < 1000) {
      throw new Error('Rate limit exceeded');
    }
    
    rateLimiter.set(key, now);
    
    // Make API call
    const response = await fetch(params.endpoint);
    return await response.json();
  },
});
```

## Best Practices

### 1. Clear Tool Names

Use descriptive, action-oriented names:

```typescript
// ❌ Bad
ToolRegistry.register({ name: 'tool1', ... });

// ✅ Good
ToolRegistry.register({ name: 'calculate_shipping_cost', ... });
```

### 2. Detailed Descriptions

Provide clear descriptions for the LLM:

```typescript
ToolRegistry.register({
  name: 'format_date',
  description: 'Format a date string into a specific format. Supports ISO 8601, US format (MM/DD/YYYY), and EU format (DD/MM/YYYY).',
  // ...
});
```

### 3. Type Safety

Use TypeScript for type-safe tools:

```typescript
interface CalculateTipParams {
  amount: number;
  percentage: number;
}

interface CalculateTipResult {
  tip: number;
  total: number;
}

ToolRegistry.register({
  name: 'calculate_tip',
  description: 'Calculate tip amount',
  parameters: {
    amount: { type: 'number', required: true },
    percentage: { type: 'number', required: true },
  },
  execute: async (params: CalculateTipParams): Promise<CalculateTipResult> => {
    const tip = params.amount * (params.percentage / 100);
    return {
      tip,
      total: params.amount + tip,
    };
  },
});
```

### 4. Error Messages

Return helpful error messages:

```typescript
ToolRegistry.register({
  name: 'validate_credit_card',
  description: 'Validate credit card number',
  parameters: {
    cardNumber: { type: 'string', required: true },
  },
  execute: async (params) => {
    if (!params.cardNumber) {
      return {
        success: false,
        error: 'Card number is required',
      };
    }
    
    if (params.cardNumber.length !== 16) {
      return {
        success: false,
        error: 'Card number must be 16 digits',
      };
    }

    const isValid = luhnCheck(params.cardNumber);
    return {
      success: isValid,
      message: isValid ? 'Valid card number' : 'Invalid card number',
    };
  },
});
```

## Examples

### Complete Tool Example

```typescript
import { ToolRegistry } from 'que-mobile-sdk';

// Register a complete tool with all features
ToolRegistry.register({
  name: 'calculate_loan_payment',
  description: 'Calculate monthly loan payment using amortization formula',
  parameters: {
    principal: {
      type: 'number',
      description: 'Loan amount',
      required: true,
    },
    annualRate: {
      type: 'number',
      description: 'Annual interest rate (as percentage, e.g., 5.5)',
      required: true,
    },
    years: {
      type: 'number',
      description: 'Loan term in years',
      required: true,
    },
  },
  execute: async (params) => {
    try {
      // Validate inputs
      if (params.principal <= 0) {
        return { success: false, error: 'Principal must be positive' };
      }
      if (params.annualRate < 0) {
        return { success: false, error: 'Interest rate cannot be negative' };
      }
      if (params.years <= 0) {
        return { success: false, error: 'Loan term must be positive' };
      }
      
      // Calculate monthly payment
      const monthlyRate = params.annualRate / 100 / 12;
      const numPayments = params.years * 12;
      
      const monthlyPayment = params.principal * 
        (monthlyRate * Math.pow(1 + monthlyRate, numPayments)) /
        (Math.pow(1 + monthlyRate, numPayments) - 1);
      
      const totalPaid = monthlyPayment * numPayments;
      const totalInterest = totalPaid - params.principal;
      
      return {
        success: true,
        monthlyPayment: Math.round(monthlyPayment * 100) / 100,
        totalPaid: Math.round(totalPaid * 100) / 100,
        totalInterest: Math.round(totalInterest * 100) / 100,
        message: `Monthly payment: $${monthlyPayment.toFixed(2)}`,
      };
    } catch (error) {
      return {
        success: false,
        error: `Calculation error: ${error.message}`,
      };
    }
  },
});

// Use the tool
const { execute } = useAgent({ apiKey: process.env.GEMINI_API_KEY! });
await execute("Calculate monthly payment for a $200,000 loan at 4.5% for 30 years");
```

### Tool with External API

```typescript
import { ToolRegistry } from 'que-mobile-sdk';

ToolRegistry.register({
  name: 'get_weather',
  description: 'Get current weather for a city',
  parameters: {
    city: { type: 'string', required: true },
    units: { type: 'string', required: false }, // 'metric' or 'imperial'
  },
  execute: async (params) => {
    try {
      const units = params.units || 'metric';
      const apiKey = process.env.WEATHER_API_KEY;
      
      const response = await fetch(
        `https://api.openweathermap.org/data/2.5/weather?q=${params.city}&units=${units}&appid=${apiKey}`
      );
      
      if (!response.ok) {
        return {
          success: false,
          error: 'City not found',
        };
      }
      
      const data = await response.json();
      
      return {
        success: true,
        temperature: data.main.temp,
        description: data.weather[0].description,
        humidity: data.main.humidity,
        windSpeed: data.wind.speed,
        message: `${data.name}: ${data.main.temp}°${units === 'metric' ? 'C' : 'F'}, ${data.weather[0].description}`,
      };
    } catch (error) {
      return {
        success: false,
        error: `Weather API error: ${error.message}`,
      };
    }
  },
});
```

## Troubleshooting

### Tool Not Found

**Problem**: Agent reports "Tool not found"

**Solutions**:
1. Verify tool is registered: `ToolRegistry.list()`
2. Check tool name spelling
3. Ensure tool was registered before agent execution

### Tool Execution Fails

**Problem**: Tool execution returns error

**Solutions**:
1. Check parameter types and values
2. Verify required parameters are provided
3. Review tool's error handling
4. Check console logs for detailed errors

### Generated Tool Doesn't Work

**Problem**: Dynamically generated tool fails

**Solutions**:
1. Provide more detailed tool description
2. Specify parameter types clearly
3. Test tool manually before using in agent
4. Check generated code for syntax errors
